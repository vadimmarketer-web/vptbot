# Известные риски

## Инфраструктурные

### NetAngels DDoS (раз в год, ~полдня)
- Прописано как форс-мажор в договоре с клиентом.
- Действия: [[../04-архитектура/runbooks/восстановление-после-DDoS]].
- Митигация: бэкапы (в работе), возможно горячий резерв у другого провайдера для крупных клиентов.

### Падение БД на VPS клиента
- Один клиент = один VPS = одна локальная БД.
- Сейчас бэкапы делаются вручную перед деплоем (см. [[../04-архитектура/runbooks/деплой-фикса]]).
- Регулярные автобэкапы — в [[../03-задачи/бэклог|бэклоге]] у Лёвы.

## Юридические

### Передача ПД клиентов клуба через мессенджер
- Формальное нарушение: ФИО+телефон в одном сообщении.
- Митигация: бот зарегистрирован на юрлицо клиента (см. [[../04-архитектура/adr/adr-004-юрлицо-клиента-в-MAX]]).
- На прямой вопрос — честно говорим, что риск есть.
- Прецедент Ильи в Крыму — попался один раз на проверке.
- 99.9% клубов хранят базы без должного шифрования; массово не ловят.
- Подробно: `smartvpt-vault/05-юр/юр-риски.md`.

### Кастомные доработки бесплатно
- Клиенты пытаются «дорабатывать продукт» — особенно когда видят карту-схему.
- Митигация: карту-схему НЕ показываем; кастомы — только за деньги.
- Принцип: «если просит ВСЕМ клубам — делаем; если только этому — допродаём».

## Продуктовые

### Раздувание UI до уровня приложения
- Кнопок и так много; каждая новая фича соблазн добавить кнопку.
- Илья и Вадим согласовали: оставаться простым ботом за 5–7к/мес.
- Сложные UI = новый ценник 50–100к/мес и другой продукт.
- Принципы: [[../03-задачи/бэклог|раздел «Принципы фильтрации фич»]].

## Будущие (когда дорастём)

### 5+ клиентов — нужен KB-бот техподдержки
- 80% вопросов будут повторяться, рутина съест команду.
- Концепция: `smartvpt-vault/07-техподдержка/концепция.md`.
- Триггер: 5 клиентов или 30+ повторяющихся вопросов в неделю.

### 10+ клиентов — найм поддержки
- Отдельный человек на общение с админами клиентов (формулировка Ильи: «сранец молодой»).

### 20+ клиентов — read-only API диагностики
- НЕ давать боту техподдержки SSH к серверам клиентов.
- Лучше: read-only API на каждом VPS для базовой диагностики (статусы, логи последних N строк, количество активных задач). Закрыто токеном, ходит только наша панель.
