# Правила vault

## Имена файлов
- русский, kebab-case или с пробелами — единообразно
- даты в формате `YYYY-MM-DD` в начале или в frontmatter
- ADR: `adr-NNN-название.md` в `09-разработка/adr/`
- Runbooks: `09-разработка/runbooks/<процесс>.md`
- Постмортемы: `YYYY-MM-DD-<инцидент>.md` в `09-разработка/инциденты/`
- Встречи: `YYYY-MM-DD-<тема>.md` в `08-встречи/`
- FAQ: `kebab-case.md` в `07-техподдержка/kb/03-faq/`

## Frontmatter (минимальный)
```yaml
---
тип: задача | решение | ADR | runbook | релиз | инцидент | встреча | faq | заметка
статус: новая | в работе | сделано | отменена
создан: YYYY-MM-DD
автор: vadim | lyova | ilya
---
```

## Связи
- `[[имя-файла]]` для ссылок между заметками
- теги `#bug`, `#release`, `#adr`, `#postmortem`, `#runbook`, `#faq`

## Что НЕ кладём
- секреты (токены, ключи) — в `~/.claude-lab/shared/secrets/`
- большие бинарники — в `99-исходники/`
- скриншоты с персональными данными клиентов — без согласия не публикуем

## Куда что кладём

| Что | Куда |
|-----|------|
| Описание продукта, фичи, продуктовый бэклог | `01-продукт/` |
| Команда, доли, регламенты | `02-команда/` |
| Информация по клубу-клиенту | `03-клиенты/<имя>.md` |
| Тарифы, бизнес-модель, биллинг | `04-финансы/` |
| Договоры, юр-риски | `05-юр/` |
| Лендинг, маркетинг-материалы | `06-маркетинг/` |
| Концепция саппорта, KB | `07-техподдержка/` |
| Протоколы встреч | `08-встречи/` |
| Дев-бэклог, архитектура, релизы, инциденты | `09-разработка/` |
| Сабпродукт KB-бота | `10-kb-бот/` |
| Транскрипты, скриншоты, сырьё | `99-исходники/` |
