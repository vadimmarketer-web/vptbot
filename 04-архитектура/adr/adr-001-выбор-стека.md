# ADR-001 — Стек: Python + Telegram + MAX + Google Sheets

## Контекст
Нужен бот для управления вводными тренировками в фитнес-клубах. Бюджет / тариф клиента 5–7к/мес — продукт должен быть **простым** и быстро воспроизводимым на новом клиенте.

## Решение
- **Язык:** Python — у Лёвы основная экспертиза, быстрая разработка, читаемость.
- **Frontend:** Telegram Bot API + MAX Bot API. Никакого мобильного приложения, никакого web UI.
- **Источник лидов:** Google Sheets, без отдельной CRM (см. [[adr-003-google-sheets-как-источник-лидов]]).
- **БД:** локальная, на VPS клиента (см. [[adr-002-vps-per-client]]).

## Последствия
- ✅ MVP запускается за дни, не недели.
- ✅ Менеджер клуба не учится новой CRM.
- ✅ Привычные мессенджеры — нулевая кривая обучения у тренеров.
- ⚠️ Все ограничения Telegram наследуем: лимиты страниц, кнопок, file size.
- ⚠️ Если перерастём 5–7к/мес — нужен будет другой стек / другой продукт.
