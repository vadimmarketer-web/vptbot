# ADR-002 — Отдельный VPS на каждого клиента

## Контекст
Альтернативы: одно shared-приложение на всех клиентов или отдельный VPS на клуб.

## Решение
**Отдельный VPS NetAngels на каждого клиента.** Барнаул и Дублёр — на разных серверах.

## Почему
- **Изоляция данных** — ПД клиентов одного клуба физически не пересекаются с данными другого. Юр. риск локализован.
- **Независимые обновления** — баг у одного клиента не валит остальных. Откат точечный.
- **Юрлицо клиента в MAX** — токен бота принадлежит клиенту, сервер с его данными — отдельный.
- **Скидка 10% при годовой оплате NetAngels** (~9к/год вместо 10–11к) — проксируется в тариф клиента, мы всё равно в плюсе.
- **Простота биллинга** — каждый клиент = свой инфра-ряд расходов, легко считать прибыль.

## Последствия
- ✅ Облако-устойчивость: падение/DDoS у одного клиента не влияет на остальных.
- ✅ Чистая модель «новый клиент → новый VPS → новый деплой».
- ⚠️ Расходы линейно растут с числом клиентов. При 20+ — нужны хотя бы единые мониторинг и read-only API диагностики.
- ⚠️ Не даём боту техподдержки SSH-доступ к серверам клиентов (огромная attack surface при росте).
