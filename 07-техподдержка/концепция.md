# KB-бот техподдержки SmartVPT — концепция

## Идея (Вадим, 27.04)

Создать одного бота техподдержки для всех клиентов SmartVPT.

```
Клиент задаёт вопрос
    ↓
Бот ищет ответ в KB (Obsidian-vault → vector store)
    ↓ confidence ≥ 0.7
Бот отвечает по KB
    
    ↓ confidence < 0.7
Эскалация в чат разработчиков
```

## Зачем
80% вопросов от клиентов = «как сделать X», не баги. Снимаем с нас рутину, не нанимаем человека.

## Когда стартовать

**НЕ сейчас (при 2 клиентах).** Триггеры запуска:
- 5+ клиентов
- ИЛИ 30+ повторяющихся вопросов в неделю

## Что делаем СЕЙЧАС

**Собираем семечки.** Каждый раз когда отвечаем клиенту в чате — копируем вопрос+ответ в `kb/03-faq/`.

К моменту 5-го клиента у нас будет 50–100 готовых заметок, выстраданных, не из головы.

## Архитектура (когда дойдём)

### Knowledge Base
```
07-техподдержка/kb/
├── 01-роли/              что может владелец/координатор/тренер
├── 02-сценарии/          как добавить тренера, координатора, зону...
├── 03-faq/               типичные вопросы из чатов клиентов
├── 04-ошибки-и-кейсы/    «дедлайн пришёл рано» → проверь, было ли распределение после 20:00
└── 05-логика-бота/       внутренняя — для эскалации, видна только саппорт-боту
```

### Технический стек
- **KB:** Markdown в Obsidian (этот vault)
- **Vector store:** Chroma или Qdrant локально
- **Embeddings:** обновление раз в день из vault
- **LLM:** Claude Haiku 4.5 (дёшево + быстро для tier-1)
- **Бот:** отдельный Telegram-бот `smartvpt-support`
- **Гейтинг:** новый user_id → запрашивает client_code (выдаёт Илья при подписании договора), без кода бот не работает

### Команды для нас (внутренние)
- `/escalations` — что бот не смог ответить за день
- `/popular` — топ-вопросов недели
- `/clients` — список клиентов и их активность

## Чего НЕ делаем

- ❌ Не даём боту SSH-доступ к серверам клиентов (огромная attack surface при 20+ клиентах)
- ❌ Не оборачиваем диагностику логов через бот (это другой продукт — нужна read-only API на каждом сервере)
- ❌ Не гонимся за 100% автоматизацией — confidence < 0.7 эскалирует **без ответа** клиенту, чтобы не врать

## Стоимость и сроки (когда дойдём)

| Что | Кто | Время |
|---|---|---|
| KB-наполнение (50–100 заметок) | Вадим + Илья | 8–12 ч |
| Бот (vector store + LLM-обвязка) | Лёва | 8–16 ч |
| Поддержка KB | Вадим | 1–2 ч/нед |
| API-стоимость | Claude Haiku | ~$20–50/мес при 5 клиентах |

## Альтернатива на старте — без LLM

Простой бот с поиском по тегам. Клиент кликает «Координатор» → видит список сценариев → выбирает «Создать координатора» → получает ответ.

10× проще LLM-варианта, 0 галлюцинаций, 0 затрат на API. Когда вопросы перестают вписываться в категории → накатываем Claude поверх.

**Эту версию можно собрать за выходные**, но всё равно нужны 50+ заметок в KB. Так что план тот же — сначала собираем сырьё.
