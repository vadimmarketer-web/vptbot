# Knowledge Base SmartVPT

## Зачем эта папка

Сюда **каждый раз когда отвечаем клиенту** в чате копируем вопрос + ответ. Когда наберётся 50+ заметок — поднимем поверх KB-бот техподдержки (см. `../концепция.md`).

## Структура (заполняется по мере)

```
kb/
├── 01-роли/              что может владелец/координатор/тренер
├── 02-сценарии/          как добавить тренера, координатора, зону...
├── 03-faq/               типичные вопросы из чатов клиентов  ← основной поток
├── 04-ошибки-и-кейсы/    «дедлайн пришёл рано» → проверь, было ли распределение после 20:00
└── 05-логика-бота/       внутренняя — для эскалации, видна только саппорт-боту
```

## Шаблон записи FAQ

```markdown
# <Краткий вопрос>

**Тема:** координатор / тренер / админ / биллинг
**Клиент:** Гравити Фит / Дублёр / общее
**Дата:** YYYY-MM-DD

## Вопрос
<как клиент сформулировал>

## Ответ
<как мы ответили>

## Контекст / причина
<почему так работает, если есть нюанс>

## Связано
- [[../04-ошибки-и-кейсы/...]]
- ссылка на код / коммит если правили

## Теги
#sla #координатор #телефоны
```

## Принцип ведения

- Записываем **сразу после ответа клиенту** в чате — не накапливаем «потом»
- Переписывать вопрос/ответ нормальным языком — потом эта же заметка станет ответом бота
- Если вопрос повторяется — увеличиваем счётчик в начале файла (или добавляем дату нового обращения)
- Раз в неделю смотрим топ — расширенные ответы выводим на сайт в FAQ
