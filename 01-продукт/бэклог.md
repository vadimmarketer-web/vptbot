# Продуктовый бэклог — принципы и стратегия

_Тактический список открытых задач — в [[../09-разработка/задачи|09-разработка/задачи]]. Здесь только принципы фильтрации фич, отклонённые предложения и стратегические идеи._

## Принципы фильтрации фич

1. **Кастом за деньги.** Если клиент просит специфичную правку — оценка по бэклогу, превышает базовый тариф 5–7к → допродажа.
2. **Не превращать в мобильку.** Цель — простой бот. Сложные UI-сценарии = новый ценник.
3. **Универсальное > клиентское.** Прежде чем делать — спроси «это нужно ВСЕМ клубам или только этому?».
4. **«Карта-схема» клиентам не показывается** — они начинают переделывать под себя. Только описание тарифа и видео-инструкции.

## Отклонённые предложения

| Что | Кто предложил | Причина отклонения |
|-----|---------------|--------------------|
| Выпадающий список ВПТ / платное ПТ | Илья | Достаточно комментария; кнопок и так много, не превращать в приложение |
| Разделение списка тренеров по зонам автоматически | Лёва | Один тренер может быть в нескольких зонах — сложно |
| SSH-доступ к серверам клиентов из бота техподдержки | — | Слишком большая attack surface при росте до 20+ клиентов |

## Идеи на будущее (стратегический горизонт)

- **Видео-туториалы по ролям** (как для админов в 1С) — снижают входной барьер для новых клиентов.
- **Read-only API для диагностики** через единую панель — необходимо до того, как KB-бот техподдержки начнёт работать с 5+ клиентами.
- **Кастомизация триал-периода / SLA** через тарифную сетку.
- **Видео-инструкции для тренеров** (общая задача проекта).
- **Кейсы с логотипами** (Гравити Фит и PHG) на лендинг — после согласования с клубами.
