# KB-бот — план MVP

## Цель MVP
Работающий Telegram-бот, в который клиент пишет вопрос, бот отвечает по KB или эскалирует в наш чат при низкой уверенности.

## Этапы

### Этап 0 — Сырьё (СЕЙЧАС, без сроков, ежедневно)
- [x] Завести структуру `07-техподдержка/kb/` (готова).
- [ ] Каждый ответ клиенту → копируем в `kb/03-faq/<тема>.md`. **Owner: Вадим + Илья (кто отвечает первым)**.
- [ ] Завести шаблон записи FAQ (см. [[сбор-FAQ]]).
- [ ] Цель — 50+ заметок к моменту разработки MVP.

### Этап 1 — Версия без LLM (выходные = 1-2 дня работы Лёвы)
Простой бот с поиском по тегам / категориям. 0 галлюцинаций, 0 затрат на API.

- [ ] Telegram-бот `smartvpt-support` у BotFather.
- [ ] Гейтинг: новый user_id → запрашивает `client_code` (выдаёт Илья при подписании договора). Без кода не работает.
- [ ] Меню: «Координатор», «Тренер», «Владелец» → список сценариев → ответ из KB.
- [ ] Команды для нас: `/escalations`, `/popular`, `/clients`.
- [ ] Деплой на тот же сервер NetAngels (отдельный systemd-юнит).

**Срок:** когда KB будет 50+ заметок — оценочно 2-4 недели от старта сбора.

### Этап 2 — Накатываем LLM поверх
Когда вопросы перестают вписываться в категории.

- [ ] Vector store: Chroma или Qdrant локально на нашем сервере.
- [ ] Embeddings: обновление раз в день из Obsidian-vault.
- [ ] LLM: **Claude Haiku 4.5** (`claude-haiku-4-5`) — дёшево + быстро для tier-1 ответов.
- [ ] При confidence < 0.7 → бот эскалирует в наш TG-чат проекта, **клиенту не отвечает**.
- [ ] Логирование всех вопросов + меток escalated/answered для последующего анализа.

**Срок:** ориентировочно через 1-2 месяца после Этапа 1, по мере роста сырья.

## Распределение работы

| Кто | Этап 0 | Этап 1 | Этап 2 |
|-----|--------|--------|--------|
| Вадим | сбор FAQ из чатов | формат, текстовка ответов, тестирование | управление KB, код-ревью текстов |
| Илья | сбор FAQ из чатов | роль/сценарии регламентирует | приоритизация эскалаций |
| Лёва | — | бот, гейтинг, deploy | vector store, LLM-обвязка, эмбеддинги |

## Стоимость (план)

| Что | Этап 1 | Этап 2 |
|---|---|---|
| Разработка | 8-16 ч Лёва | +8-16 ч Лёва |
| Поддержка KB | 1-2 ч/нед Вадим | 1-2 ч/нед Вадим |
| API Claude Haiku | $0 | ~$20-50/мес при 5 клиентах |

## Что считаем успехом MVP
- 80% вопросов клиентов получают ответ без эскалации.
- Среднее время ответа на эскалацию ≤ 1 рабочий день (по SLA).
- Команда тратит на саппорт < 30 мин/день вместо 2-3 часов сейчас.
