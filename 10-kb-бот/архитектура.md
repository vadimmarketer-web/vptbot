# Архитектура KB-бота

## Стек

| Компонент | Этап 1 (без LLM) | Этап 2 (с LLM) |
|---|---|---|
| Бот | Telegram Bot API, Python (как основной SmartVPT) | то же |
| KB | Markdown в Obsidian (`07-техподдержка/kb/`) | то же + индексирование |
| Поиск | Поиск по тегам / категориям | Vector store (Chroma или Qdrant) |
| Embeddings | — | Обновление раз в день из vault |
| LLM | — | **Claude Haiku 4.5** (`claude-haiku-4-5`) |
| Гейтинг | client_code | то же |
| Хостинг | NetAngels (отдельный VPS или общий с одним из клубных ботов? — решить) | то же |

## KB-структура (в `07-техподдержка/kb/`)
```
kb/
├── 01-роли/              что может владелец/координатор/тренер
├── 02-сценарии/          как добавить тренера, координатора, зону...
├── 03-faq/               типичные вопросы из чатов клиентов
├── 04-ошибки-и-кейсы/    «дедлайн пришёл рано» → проверь, было ли распределение после 20:00
└── 05-логика-бота/       внутренняя — для эскалации, видна только саппорт-боту
```

## Поток (Этап 2 — с LLM)
```
[Клиент в TG]
    │
    ▼
[smartvpt-support бот]
    │
    ├─→ user_id известен? нет → запрос client_code → не валиден → отказ
    │
    ▼
[Поиск в vector store]  (top-k чанков из KB)
    │
    ▼
[Claude Haiku 4.5: ответь по контексту, верни confidence]
    │
    ├─→ confidence ≥ 0.7  →  Ответ клиенту + лог
    └─→ confidence < 0.7  →  Эскалация в наш TG-чат + лог. Клиенту: «передал команде, ответим в течение N часов».
```

## Гейтинг по client_code
- Илья при подписании договора выдаёт клиенту уникальный `client_code` (строка типа `BARNAUL-2026-A`).
- Новый пользователь TG → бот спрашивает код → проверяет в БД → привязывает user_id к клубу.
- Без валидного кода — отказ. Не публичный бот.

## Внутренние команды (для нашей команды)
- `/escalations` — что бот не смог ответить за день, со ссылкой на чат.
- `/popular` — топ вопросов недели (для пополнения KB).
- `/clients` — список клиентов и их активность (число вопросов / эскалаций).

## Безопасность
- ❌ **НЕТ SSH-доступа** к серверам клиентов из бота. Нужна диагностика — отдельный read-only API на каждом VPS, закрытый токеном (см. [[../04-архитектура/архитектура|архитектура SmartVPT]]).
- ❌ Бот не выдаёт техническую информацию о других клиентах. Только своя KB-зона по client_code.
- ✅ Все ответы логируются с user_id + client_code → разбор инцидентов.

## Где живёт код
- В этом же репо `vptbot`? Или отдельный репозиторий? — **решить с Лёвой**.
- Аргумент за общий репо: один деплой-флоу, общие утилиты.
- Аргумент за отдельный: KB-бот живёт на одном сервере, а SmartVPT — по серверу на клиента; бинарно разные модели деплоя.
